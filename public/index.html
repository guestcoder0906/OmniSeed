📖 Advanced Block Pattern Syntax Guide

The Unified Engine uses a powerful voxel-logic parser to match complex 3D structures. Patterns are evaluated within each chunk in the search radius.

1. Exact Voxel Offsets (Disconnected)
Syntax: [X,Y,Z]+[X,Y,Z]...
Example: [0,0,0]+[3,0,0]+[1,-1,0]+[2,-1,0] (Smiley Face pattern). Blocks are placed exactly at these relative offsets.
2. 3D Geometric Volumes (Arrays)
Syntax: WxHxD
Example: 1x10x1 (A 10-block high pillar). Supports variable heights with ranges like 1x10-15x1.
3. Composite/Nested Logic (Composite Dimensions)
Syntax: [X,Y,Z(label)=Shape]+[X,Y,Z=Shape]
Example: [0,0,0(up)=1x2x1]+[0,-1,0(down)=(center: 3x1x1)]
Combines exact relative positioning with volume templates. Labels like (up) specify orientation or context.
4. Natural Generation Veins
Syntax: N+ or vein:N+
Example: 8+ (Finds ore veins of 8 blocks or more). Use this for realistic ore blobs.
5. Modifiers & Logical States
Appending :dug or :filled allows matching hollow structures (like rooms) vs solid masses. (center: ...) allows auto-centering sub-shapes on the parent coordinate.
